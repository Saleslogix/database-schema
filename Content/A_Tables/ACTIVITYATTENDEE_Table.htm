ACTIVITYATTENDEE Table

Added in Version 8.1

The ActivityATTENDEE table stores additional contacts and leads for Activities. This table was added for multiple attendees per activity functionality.

 

  • While this table does not have a field defined as a primary key, the ACTIVITYATTENDEEID field should be unique and considered a primary key.

  • EntityID is used as a foreign key to hold a ContactID, LeadID or custom entity ID.
