Understanding Database Triggers

Added in Version 8.1

Database triggers are designed to improve performance and capture all database changes regardless of where the change occurred. This means that changes made during a batch update will capture the change for each record that was updated. Database trigger information is stored in the DB_OBJECTDEFINITION table so that the triggers are sent out to remote databases during synchronization.

In Infor version 8.1, the CONTACTSYNCTRIGGER Provider trigger was removed from the SLXTRIGGERS table and replaced with multiple database triggers. Additional database triggers were added for integrations introduced in version 8.1 and later.

 GLOBALCHANGETRACKING_IDENTITY is listed as a trigger in the bundle, but is not actually a trigger. The GLOBALCHANGETRACKING_IDENTITY is a SQL specific function to create an identity column in the GLOBALCHANGETRACKING table in host and remote databases.
