Disabling and Enabling Database Triggers

For performance reasons, you may want to disable database triggers in certain instances:

  • If your implementation does not include any integrations (Outlook sync, Google Sync, or X3)
  • If your implementation includes one or more integrations and you need to perform a large import or large batch updates that do not involve data that is synchronized between Infor and the integration.

If your implementation does not include any integrations then the triggers can remain disabled. If your implementation includes any integrations, then you must enable the triggers once the import or batch update is complete.

If you disabled database triggers prior to implementing any integrations and now plan to implement one or more integrations, you must do the following:

  1. Enable the database triggers using the scripts listed for your database type.
  2. Run the SLXConversionUtility.exe.
    For steps see the Upgrading to Infor CRM v8.2.pdf.
  3. Perform the steps required for implementing the integration.
    These steps are specific to the integration. Please refer to the Infor CRM Web Client help or documentation provided with the integration.

 The ACCOUNT trigger is only required for X3. If your implementation does not include X3, this trigger can remain disabled. All other triggers are required for Outlook sync and Google Sync.
SQL

To disable database triggers

  • Run the following scripts:
    • ALTER TABLE SYSDBA.ACCOUNT DISABLE TRIGGER ALL (X3 integration only)
    • ALTER TABLE SYSDBA.ACTIVITY DISABLE TRIGGER ALL
    • ALTER TABLE SYSDBA.ACTIVITYATTENDEE DISABLE TRIGGER ACTIVITYATTENDEE_INTEGRATION_CHANGE
    • ALTER TABLE SYSDBA.ADHOCGROUP DISABLE TRIGGER ALL
    • ALTER TABLE SYSDBA.CONTACT DISABLE TRIGGER ALL
    • ALTER TABLE SYSDBA.USER_ACTIVITY DISABLE TRIGGER ALL

To enable database triggers

  • Run the following scripts:
    • ALTER TABLE SYSDBA.ACCOUNT ENABLE TRIGGER ALL (X3 integration only)
    • ALTER TABLE SYSDBA.ACTIVITY ENABLE TRIGGER ALL
    • ALTER TABLE SYSDBA.ACTIVITYATTENDEE ENABLE TRIGGER ACTIVITYATTENDEE_INTEGRATION_CHANGE
    • ALTER TABLE SYSDBA.ADHOCGROUP ENABLE TRIGGER ALL
    • ALTER TABLE SYSDBA.CONTACT ENABLE TRIGGER ALL
    • ALTER TABLE SYSDBA.USER_ACTIVITY ENABLE TRIGGER ALL
Oracle

To disable database triggers

  • Run the following scripts:
    • ALTER TABLE SYSDBA.ACCOUNT DISABLE ALL TRIGGERS (X3 integration only)
    • ALTER TABLE SYSDBA.ACTIVITY DISABLE ALL TRIGGERS
    • ALTER TABLE SYSDBA.ACTIVITYATTENDEE DISABLE TRIGGER ACTIVITYATTENDEE_INTEGRATION_CHANGE
    • ALTER TABLE SYSDBA.ADHOCGROUP DISABLE ALL TRIGGERS
    • ALTER TABLE SYSDBA.CONTACT DISABLE ALL TRIGGERS
    • ALTER TABLE SYSDBA.USER_ACTIVITY DISABLE ALL TRIGGERS

To enable database triggers

  • Run the following scripts:
    • ALTER TABLE SYSDBA.ACCOUNT ENABLE ALL TRIGGERS (X3 integration only)
    • ALTER TABLE SYSDBA.ACTIVITY ENABLE ALL TRIGGERS
    • ALTER TABLE SYSDBA.ACTIVITYATTENDEE ENABLE TRIGGER ACTIVITYATTENDEE_INTEGRATION_CHANGE
    • ALTER TABLE SYSDBA.ADHOCGROUP ENABLE ALL TRIGGERS
    • ALTER TABLE SYSDBA.CONTACT ENABLE ALL TRIGGERS
    • ALTER TABLE SYSDBA.USER_ACTIVITY ENABLE ALL TRIGGERS
