SLXSQLDEFAULT Table

Added in Version 7.0

The SLXSQLDEFAULT table stores configuration entries for default values. This table contains information including, but not limited to, table name, field name and default type.

Table Relationships

The SLXSQLDEFAULT table has a (Many) to (Zero or One) relationship with the SLXOLEDBPLUGIN table.

  • CreateUser and ModifyUser are foreign keys referencing UserID in the USERINFO table.

  • If an index entry appears in multiple table columns, then it is a composite index. A composite index requires multiple columns in order to create a unique index.
