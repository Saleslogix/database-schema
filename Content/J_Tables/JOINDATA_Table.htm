JOINDATA Table

The JoinData table stores global join definitions. When you join tables, a link is created between two tables using a field that exists in both tables. Global joins are accessed using the Global Join Manager in the Administrator, Architect, and Query Builder.

  • CreateUser and ModifyUser are foreign keys referencing UserID in the USERINFO table.

  • If an index entry appears in multiple table columns, then it is a composite index. A composite index requires multiple columns in order to create a unique index.
