Database Help System

Infor CRM supports Microsoft SQL Server and Oracle relational database management systems. By definition, relational databases contain tables (rows and columns) of information. The structure of the database is defined by how these tables relate to each other. Since information is retrieved based on table relationships, it is important to understand these relationships before customizing Infor CRM or running database queries.

Saleslogix is now Infor CRM. Look for the new branding in this release and future releases.

All references to the CRM Core product, CRM Clients, Web Client, Windows Client, Administrator, and database refer to Saleslogix versions between 8.0 and 8.1 update 03 unless otherwise specified.

The Database Help system documents all the tables and fields in the Saleslogix Evaluation database for the base product.

Using the Database Manager Help

  • Use the Help Table of Contents to view the properties for each table.

  • Use the following legend for each table:PK = Primary Key; FK = Foreign Key; CA= Clustered Index;AK= Unique Index;= Non-unique Index

  • In most tables, foreign key (FK) relationships are noted or the field names are in blue text allowing you to jump to the related table.

  • Use the Search tab to search for a field name. Type the field name in the keyword to find box, click List Topics and the results will contain all of the tables/topics where the field name appears. As an example, if you search on EntityID, the results are the ADDRESS table and ADHOCGROUP table.
