Legend

PK = Primary Key

FK = Foreign Key - indicates a parent/child relationship to another table.  The customary constraints associated with a foreign key do not apply.

CA = Clustered Index

AK = Unique Index

IE = Non-unique Index

(One) to (One) = For a single record in one table there is only one related record in another table. Bold face indicates the relationship is defined in the Application Architect entity model. Otherwise, the relationship is only implied. Add the relationship in Application Architect if you want to access the related entities through the object hierarchy.

(One) to (Many) = For a single record in one table there are one or more related records in another table. Bold face indicates the relationship is defined in the Application Architect entity model. Otherwise, the relationship is only implied. Add the relationship in Application Architect if you want to access the related entities through the object hierarchy.

(Many) to (Many) = There are many records in one table that relate to many records in another table. Bold face indicates the relationship is defined in the Application Architect entity model. Otherwise, the relationship is only implied. Add the relationship in Application Architect if you want to access the related entities through the object hierarchy.
