PROCESS Table

The Process table stores information about Saleslogix processes assigned to Saleslogix contacts. Processes are created in Architect as plugins, and then released to users to assign to contacts. The stored information includes the process name, family, contact target ID and name, the status of the process, the start date, the date of the next action, if an activity is part of the process, has the process been suspended, the process node, date of the last action, the details that appear in the Summary tab, the process plugin ID the process is based on, and the ID of the user starting the process.
